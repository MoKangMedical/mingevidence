# 明证知识底座蓝图

## 目标

中国版 OpenEvidence 不能只做“问答式搜索”，而要形成三层知识底座：

1. 中国正式证据层
   - 中国指南
   - NMPA 说明书与监管文件
   - 医保与本地可及性规则
2. 国际监管与研究层
   - FDA 标签
   - PubMed 摘要级证据
   - 国际指南与关键研究
3. 中国人群分层与全病程信号层
   - 风险预测
   - 早期预警
   - 诊断与分型
   - 治疗选择
   - 疗效监测
   - 不良事件监测
   - 复发/再入院随访

## 当前实现

### 1. 正式证据 ingestion

- `scripts/ingest-evidence.mjs`
- `data/sources/china-guidelines.sources.json`
- `data/sources/drug-labels.sources.json`
- `data/sources/fda-labels.sources.json`

当前支持：

- HTML 官方页面采集
- NMPA PDF 抽取
- openFDA label API 拉取
- 统一输出 `evidence-records.json` 与 `evidence-chunks.json`

### 2. 搜索排序

- `src/lib/search-service.ts`

当前排序逻辑：

- 中国指南优先
- NMPA / FDA 标签作为监管边界层
- PubMed 摘要作为国际补充层
- chunk 级命中后再 rerank

### 3. 中国人群分层信号

- `scripts/sync-population-signals.mjs`
- `data/sources/population-signals.sources.json`
- `src/lib/population-signal-schema.ts`

当前输出：

- `data/normalized/population-signals.json`

该文件面向后续两个方向：

1. DeepConsult 的中国人群解释层
2. 疾病预测/预警与全病程监测模块
