# 明证系统架构

## 目标

按照 OpenEvidence 的模式，先做“证据搜索 + DeepConsult”，再扩展到医院知识底座。

## 首版分层

### 1. 采集层

- 中国指南与专家共识
- 药监审批和说明书
- 医保目录与支付规则
- PubMed / 国际期刊元数据
- 后续加入院内路径和知识库

### 2. 处理层

- 文档清洗
- 章节切分
- 元数据标准化
- 指南和研究类型标注
- 时间戳和版本管理

### 3. 检索层

- BM25/关键词检索
- 向量检索
- 指南优先排序
- 中国本地内容加权
- 新近研究补充排序

### 4. 推理层

- 临床问题分类
- 证据聚合
- 回答生成
- 引文对齐校验
- 风险问题分级

### 5. 治理层

- 拒答与降级规则
- 高风险问题人工复核
- 输出日志
- 医生纠错反馈
- 版本审计

## 首版技术建议

- 前端：Next.js
- API：Next Route Handlers
- 搜索：先 mock，后接 PostgreSQL + pgvector 或 Elasticsearch
- 模型：先保留模型适配层，不绑定单一供应商
- 医学治理：单独设计规则引擎，不混进 prompt
